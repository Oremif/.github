# Governance

Oremif is a maintainer-led open source organization.

## Maintainer model

- Oremif is currently maintained primarily by `@devcrocod`.
- Repository maintainers are responsible for roadmap decisions, release management,
  API acceptance, and final merge decisions.
- Repository-specific governance documents, if present, take precedence over this
  default file.

## How decisions are made

- Small changes are usually handled through normal issue and pull request review.
- Large features, breaking API changes, and major architectural changes should be
  discussed in an issue before implementation.
- Maintainers aim to balance correctness, API stability, maintainability, and user
  impact when making decisions.

## Project scope

Oremif focuses on Kotlin Multiplatform libraries for mathematics, data science, AI,
and related developer tooling. Not every proposed feature fits the scope of every
repository, even if it is technically valid.

## Contributions

Contributions are welcome, but acceptance is not automatic. A pull request may be
declined if it conflicts with project scope, design goals, maintenance capacity, or
long-term API direction.
