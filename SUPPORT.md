# Support

This repository provides the default support guidance for Oremif projects. If a
repository has its own `SUPPORT.md`, that repository-specific guidance takes
precedence over this file.

## Before asking for help

- Read the repository README and project documentation.
- Search existing issues for the same question or error.
- Check examples, release notes, and changelogs if available.

## Where to ask

- Use GitHub Issues for reproducible bugs and feature requests.
- Use the issue templates when available.
- For security issues, follow the instructions in `SECURITY.md` and do not post
  sensitive details publicly.

## What to include

When asking for help, include enough context to reproduce the problem:

- package version or commit SHA;
- Kotlin version and target platform;
- minimal sample code;
- exact error message or stack trace;
- expected behavior and actual behavior.

## Response expectations

Oremif projects are maintained on a best-effort basis by a solo maintainer. Support
is community-style and response times may vary.
